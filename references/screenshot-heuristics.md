# Screenshot Heuristics

这个参考文件用于帮助判断哪些字幕句子值得配图，重点面向屏幕录制类视频。

## 目标

截图的作用是降低读者复现操作的成本，而不是装饰版面。

## 高优先级画面

优先截图这些信息密度高、视觉依赖强的位置：

- 鼠标点击、菜单展开、页面跳转、表单填写
- 代码编辑器中出现关键代码块、文件名、报错行
- 终端命令执行后出现结果、警告、错误
- 设置面板、权限弹窗、安装向导、下载页面
- 前后状态变化明显的瞬间

## 触发词

字幕中出现这些表达时，通常值得考虑截图：

- 这里 / 这儿 / 这一项 / 这个按钮 / 这个页面
- 可以看到 / 如下 / 打开 / 点击 / 输入 / 选择 / 切换
- GitHub / URL / API / endpoint / localhost / 地址
- 报错 / 日志 / 输出 / 配置 / 参数

## 去重规则

- 同一个静态画面附近，不要连续插入多张图
- 同一个操作流中，优先保留“动作发生时”那一张，而不是每一句都截
- 如果一张图已经足够说明上下两句，可只在第二句末尾插入一次

## 屏幕录制特化建议

- 优先展示操作对象，而不是空白过渡帧
- 如果字幕时间点刚好落在切场瞬间，截图更适合取该句末尾秒
- 对代码讲解，尽量截到完整的代码片段或文件区域
- 对 UI 操作，尽量截到按钮、弹窗或结果区域同时可见的画面

## 示例判断

适合截图：

- “这里我们点一下右上角的 Deploy。”
- “运行后你会看到终端里出现这个报错。”
- “把这个地址复制到浏览器里打开。”
- “这里的 `vite.config.ts` 需要改一下。”

通常不需要截图：

- “接下来我再补充一个背景知识。”
- “我们稍后还会回到这里继续讲。”
- “这个概念本身并不复杂。”
