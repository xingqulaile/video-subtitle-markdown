# 项目介绍文案

## 版本一：适合 GitHub README 或项目简介

`video-subtitle-markdown` 是一个给 Codex 使用的 skill。

它可以把“视频 + `.srt` 字幕”自动整理成带本地截图的图文 Markdown。它不只是把字幕转成笔记，还会识别哪些句子需要视觉辅助，在合适位置插入截图占位符，再调用 `ffmpeg` 自动截图并回填到 Markdown 里。

比较适合这些场景：

- 屏幕录制教程
- 软件操作演示
- 网页 walkthrough
- 代码讲解视频
- 团队内部 SOP 沉淀

最终产物不是一份干巴巴的字幕稿，而是一篇可以直接阅读、分享和复用的图文记录。

## 版本二：适合朋友圈 / 社媒短帖

我最近做了一个很实用的 Codex skill：`video-subtitle-markdown`。

它能把“视频 + 字幕文件”直接整理成图文 Markdown，不只是转写字幕，还会自动判断关键步骤，在合适的位置调用 `ffmpeg` 截图，把图片插回文档里。

特别适合整理：

- 软件操作演示
- 屏幕录制教程
- 代码讲解视频
- 团队知识库内容

以前看完视频还得手动截图、手动排版、手动补文档。现在可以先录视频，再把内容自动沉淀成一篇图文笔记，省很多重复劳动。

## 版本三：适合发技术社区

做这个 skill 的出发点很简单：

很多教程内容最初都来自视频，但视频并不适合检索、复习和复用。字幕虽然能导出成文本，可如果没有截图，很多“这里点一下”“这个按钮”“这个报错”“这个配置项”的上下文还是会丢。

所以我做了一个 Codex skill，专门处理这类内容：

1. 读取 `.srt`
2. 整理为 Markdown
3. 在真正需要视觉辅助的句子后插入截图占位符
4. 调用 `ffmpeg` 自动截图
5. 生成最终图文版 Markdown

这样最终沉淀下来的不是“字幕文本”，而是一篇真正可阅读、可复现、可归档的图文文档。
